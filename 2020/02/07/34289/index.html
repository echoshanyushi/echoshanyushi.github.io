博客自制指南

Posted on 2020-02-07 | Post modified: 2020-02-12 | In Blog DIY Guide | |

本篇文章介绍本博客，即基于Github Pages并采用Hexo框架和Next主题的博客自制指南。

前言


博主是个非编程类工科生，自己做博客主要是为了玩和练笔。要制作这个博客，不需要会编程，只需要了解一些简单的语法知识。网上的教程很多，但有些总有一点小错漏，让编程小白抓耳挠腮。博主在程序猿夏同学的帮助下，断断续续两天之内顺利布置好了这个博客。现在根据亲身经历写成这篇指南，一为纪念，二为反馈，希望有缘的你也能在自制博客的过程中感受到布置新家一样的乐趣。

指南框架


本文主要将博客自制过程分为三步：

  1. Github Pages博客搭建

  2. Next主题美化

  3. Markdown博文书写

Github Pages博客搭建


这里需要三个东西：

  1. IntelliJ IDEA

  2. Github

  3. Hexo

IntelliJ IDEA

IntelliJ IDEA（以下简称IDEA）是被夏同学强烈推荐的，程序猿必备的管理工具。自制博客过程中各种文件类型的代码修改，以及后面写.md的博文都可以用这个软件，手动修改任何东西都会自动保存，左边的小箭头能够指示.md文件的段落，而且界面五彩缤纷的非常友好。

Github

Github Pages是Github提供的博客服务，在Github中创建一个特定格式的Repository，Github Pages就会将其中的内容生成一个网页，也就是自己的博客展示出来。步骤如下：

  1. 注册Github账号，比如echoshanyushi

  2. 新建一个Repository，命名为echoshanyushi.github.io，勾选初始化一个README.md，注意必须为账号.github.io，才会被Github Pages拿去生成网页

  3. 随便在README.md里面写点啥，比如“Hello World!”

  4. 打开网页：https://echoshanyushi.github.io/，就可以看到“Hello World!”了，自己的网页雏形，是不是很激动！

如果直接用这个README.md来写博文也是可以的，但是这也显得太极简了吧，于是我们需要一个好看的框架来承载我们好看的心灵，哦不，好看的博客。

Hexo

Hexo是一个博客框架，它可以嵌入好看的主题，连同我们写的博文.md文件一起生成本地的.html文件，最后推给Github去生成公开网页。步骤如下：

  1. 安装Node.js：https://nodejs.org/en/

  2. 安装Git：https://git-scm.com/

  3. 安装Hexo：win+R打开cmd窗口，输入npm install -g hexo-cli并回车

  4. 创建本地博客

    • 在E盘下创建文件夹MyBlog并打开
    • 右键选择Git Bash Here，打开之后的路径就是(E:\MyBlog)
    • 输入hexo ini将这个文件夹初始化为博客文件夹
    • 输入npm install安装依赖包
    • 输入hexo g或者hexo generate生成静态.html网页文件
    • 输入hexo s或者hexo server将本地网页放到本地服务器：http://localhost:4000/，打开链接可以看到Hexo默认的博客主题效果以及一篇自带的Hello World的博文
    • 回到Git Bash，按Ctrl+C结束，此时本地服务器链接已不可访问了
       

  5. 修改配置文件

    • 在E盘下创建文件夹MyBlog并打开
    • 到Github的Repository中，Clone or download对应的链接：https://github.com/echoshanyushi/echoshanyushi.github.io.git
    • 用IDEA打开站点配置文件(E:\MyBlog_config.yml)
    • Ctrl+F找到# Deployment，填入如下代码，注意语法很严格，type这类副键前必须是一个Tab，git这类键值前必须有一个空格，天知道Tab这个BUG我找了多久= =

  6. 将本地博客部署到Github上

    • 回到Git Bash，输入npm install hexo-deployer-git --save，安装hexo-deployer-git
    • 输入hexo d或者hexo deploy
    • 得到INFO Deploy done: git即为部署成功，之前Github中的README.md会被覆盖掉
    • 打开网页：https://echoshanyushi.github.io/，就可以看到之前本地服务器所看到的博客，大功告成！
       

  7. 发布博文流程

    • 在MyBlog文件夹中选择Git Bash
    • 输入hexo n "My First Post"或者hexo new "My First Post"
    • 在(E:\MyBlog\source_posts)路径下，会有一个My-First-Post.md的博文文件，编辑这个文件并保存
    • 输入hexo s，查看本地博客效果，按Ctrl+C结束
    • 输入hexo cl或者hexo clean，清空生成过的静态.html文件和缓存数据，比如(E:\MyBlog\public)文件夹就没有了
    • 输入hexo g，生成新的静态.html文件
    • 输入hexo d，部署到Github上
    • 打开网页：https://echoshanyushi.github.io/，查看更新后的博客

至此Github Pages博客搭建就完成了，但此时的博客界面非常简单，并不是很好看也没有个性化，因此接下来进入Next主题美化部分。

Next主题美化


这里开始就都是一些琐碎的设置了，主要为修改站点配置文件(E:\MyBlog\_config.yml)和主题配置文件(E:\MyBlog\themes\next\_config.yml)以及其他文件，根据个人喜好进行个性化的博客主题美化。所有文件的根目录都在(E:\MyBlog)下，博主采用的是Next主题，本博客的主题配置步骤如下：

  1. 克隆Next主题，回到Git Bash，输入git clone https://github.com/iissnan/hexo-theme-next themes/next，此时themes文件夹下面就有一个next文件夹了

  2. 设置博客主题为Next主题

    • 打开主题配置文件，找到scheme: Gemini，去掉前面的注释符号#，在未选的主题前加上#

    • 若成功，则以后新建博文之后，在.md文件的上方添加categories: 分类1，tags: 标签1或tags: [标签1, 标签1]就可以将博文自动归入对应的分类和添加对应的标签了
